# Granting Admin Permissions

**Important Note:** FundGen has introduced a new, more flexible permission system based on **Custom Roles**. While the old method of granting 'Admin' permissions via a single toggle still functions for legacy purposes, we highly recommend using the new Custom Roles feature for granular control over user access. This guide focuses on the legacy method, but we encourage you to explore [Creating and Managing Custom Roles](/articles/campaign-management/creating-and-managing-custom-roles) for advanced permission management.

## Who is this guide for?

- Campaign Managers
- Strategy Team
- Select Key Ambassadors

## Prerequisites

- You must be logged in as a user with **Admin Permissions**.
- The user receiving the permission must already exist in the system (meaning, they have taken a goal in the campaign).

## Steps to Grant Admin Permissions (Legacy Method)

1.  **Log in to the Admin Interface** (via computer/mobile) – [https://admin.givingl.ink/dashboard](https://admin.givingl.ink/dashboard)
2.  In the main toolbar, select the **Contacts** menu.
3.  From the menu, enter the **Manage Ambassadors** tab.
4.  After entering the tab, scroll down slightly on the page that opens.
5.  The page will display the list of campaign ambassadors, including all participants who have taken a campaign goal.

    To the left of each ambassador, you will find an icon resembling a **figure with a plus sign (+)**. Clicking this icon grants the ambassador admin permissions. Once the icon turns **green** after clicking, it indicates that the action has been successfully completed.

    *For more granular control, please refer to the new guide on [Creating and Managing Custom Roles](/articles/campaign-management/creating-and-managing-custom-roles) and [Assigning Custom Roles to Users](/articles/campaign-management/assigning-custom-roles-to-users).*

# נתינת הרשאת ניהול

**הערה חשובה:** FundGen הציגה מערכת הרשאות חדשה וגמישה יותר המבוססת על **תפקידים מותאמים אישית**. בעוד שהשיטה הישנה למתן הרשאות 'מנהל' באמצעות כפתור בודד עדיין פועלת למטרות תאימות לאחור, אנו ממליצים בחום להשתמש בתכונת התפקידים המותאמים אישית החדשה לשליטה מדויקת יותר בגישת המשתמשים. מדריך זה מתמקד בשיטה הישנה, אך אנו מעודדים אותך לבדוק את [יצירה וניהול תפקידים מותאמים אישית](/articles/campaign-management/creating-and-managing-custom-roles) לניהול הרשאות מתקדם.

**מטרת הפעולה**

הרשאת ניהול מאפשרת למשתמש נוסף לצפות, לערוך ולנהל נתונים במערכת בהתאם להרשאות שנקבעו.

**למי מיועד המדריך**

● מנהלי הקמפיין
● אנשי אסטרטגיה
● שגרירים משמעותיים בודדים

**תנאים מקדימים**

● עליך להיות מחובר עם **משתמש בעל הרשאת ניהול**.
● המשתמש שיקבל הרשאה חייב להיות **קיים במערכת (כלומר, שלקח יעד בקמפיין)**.

# שלבים למתן הרשאת ניהול (שיטה ישנה)

1.  **כניסה לממשק הניהול** (באמצעות מחשב/נייד) – [https://admin.givingl.ink/dashboard](https://admin.givingl.ink/dashboard)
2.  בסרגל הכלים הראשי, יש לבחור בתפריט **אנשי קשר**.
3.  מתוך התפריט, יש להיכנס ללשונית **ניהול שגרירים**.
4.  לאחר הכניסה ללשונית, יש לגלול מעט למטה בעמוד שנפתח.
5.  בעמוד תוצג **רשימת השגרירים של הקמפיין**, הכוללת את כלל המשתתפים שלקחו יעד בקמפיין.

    בצד שמאל של כל שגריר, יופיע אייקון בצורת **דמות עם סימן פלוס (+)**. לחיצה על אייקון זה מעניקה לשגריר הרשאת ניהול. לאחר הלחיצה, כשהאייקון נצבע **בירוק**, הדבר מציין שהפעולה בוצעה בהצלחה.

    *לשליטה מדויקת יותר, אנא עיין במדריכים החדשים על [יצירה וניהול תפקידים מותאמים אישית](/articles/campaign-management/creating-and-managing-custom-roles) ו[הקצאת תפקידים מותאמים אישית למשתמשים](/articles/campaign-management/assigning-custom-roles-to-users).*
